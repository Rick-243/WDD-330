    Notes:

    - Chapter 5: - All objects are mutable at any time when a program is running.
    This means that its properties and methods can be changed or removed, and new
    properties and methods can be added to the object, even if it was declared using const.
    To create an object literal, simply enter a pair of curly braces. The following example
    creates an empty object that is assigned to the variable spiderman :
    const spiderman = {};
    The above example is the preferred method of creating an object, and is an object literal notation.
    ES6 provided a shorthand method of creating objects if a property key is the same as a variable name
    that the property value is assigned to:
    const name = 'Iron Man';
    const realName = 'Tony Stark';
    // long way
    const ironMan = { name: name, realName: realName };
    // short ES6 way
    const ironMan = { name, realName };
    Dot notation is much more common, but bracket notation has a few advantages: it's the only way to access
    nonstandard property and method names that donâ€™t follow the variable naming rules. It also lets you evaluate
    an expression and use it as the property key:
    const superman = {
    name: 'Superman',
    'real name': 'Clark Kent',
    height: 75,
    weight: 235,
    hero: true,
    villain: false,
    allies: ['Batman','Supergirl','Superboy'],
    fly() {
    return 'Up, up and away!';
    }
    };
    superman["real" + " " + "name"] // the property is built using string concatenation
    << "Clark Kent"
    The expression 'real name' was used as the property key.
